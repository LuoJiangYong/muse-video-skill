# 角色一致性策略 — Character Consistency

> **定位**：AI 生图中最难的问题——同一个角色在不同镜头/场景/姿势下保持外观一致。本文档诚实评估每种策略的能力边界和局限性。
> **宪法约束**：在 Phase 3（视觉开发）时必须诚实告知用户：AI 角色一致性最多达到 70-80%。不要承诺 100%。

---

## 问题定义

### 为什么角色一致性是 AI 生图的「圣杯难题」

1. **扩散模型的本质是随机采样**——同样的 prompt + seed 在相同分辨率下能产生相同的图，但一旦改变 pose/camera/lighting，即使 seed 相同，角色也会变化
2. **没有「角色 ID」这个概念**——模型不理解「这是同一个角色」，它只理解「短发蓝眼女性」
3. **跨分辨率/跨画幅**——改变分辨率会改变 latent space，进一步加剧不一致

### 一致性评价维度

| 维度 | 说明 | 达标标准 |
|------|------|---------|
| **面部特征** | 五官/脸型/肤色 | 同一人脸的辨识度 |
| **体型** | 身高/胖瘦/肩宽 | 比例一致 |
| **发型发色** | 长度/颜色/质感 | 视觉一致 |
| **服装** | 款式/颜色/材质 | 精确到单品 |
| **配饰/标记** | 纹身/疤痕/眼镜 | 精确复制 |

## 策略矩阵（能力边界诚实标注）

### 策略 1：种子固定（Seed Fixing）

| 维度 | 评估 |
|------|------|
| **原理** | 相同 seed + 相同 prompt + 相同参数 = 相同图片。改变 pose/lens/camera → seed 失效 |
| **成本** | 零 |
| **面部一致性** | ❌ 差（无法跨 prompt 保持） |
| **体型一致性** | ❌ 差 |
| **发型一致性** | ❌ 差 |
| **服装一致性** | ❌ 差 |
| **适用场景** | 仅适用于同一构图的重跑（如调整 CFG/steps），不是角色一致性方案 |
| **诚实评级** | ⭐ 这不是角色一致性策略，是重跑同一张图的策略 |

### 策略 2：IP-Adapter（ComfyUI）

| 维度 | 评估 |
|------|------|
| **原理** | 用一张参考图（角色定妆照）作为「图像 prompt」，影响生成结果的视觉风格 |
| **成本** | 低（需要 1 张高质量参考图） |
| **面部一致性** | ⭐⭐⭐ 中-高（取决于参考图质量 + IP-Adapter 权重） |
| **体型一致性** | ⭐⭐ 中（IP-Adapter 对面部最有效，体型受 prompt 影响较大） |
| **发型一致性** | ⭐⭐⭐ 中-高 |
| **服装一致性** | ⭐ 低（IP-Adapter 主要影响风格，不是精确复制物品） |
| **适用场景** | 最推荐的通用方案。适合：同一角色多个镜头，同一角色换装 |
| **参数建议** | IP-Adapter weight: 0.6-0.8（太高 → 过度拟合参考图，太低 → 效果不明显） |
| **诚实评级** | ⭐⭐⭐⭐ 推荐。是目前 AI 角色一致性的最佳性价比方案 |

### 策略 3：FaceID / InstantID（ComfyUI）

| 维度 | 评估 |
|------|------|
| **原理** | 提取参考人物的面部 embedding，在生成过程中「注入」目标面部特征 |
| **成本** | 低-中（需要面部 embedding 模型 + 参考图） |
| **面部一致性** | ⭐⭐⭐⭐ 高（对五官精确度远高于 IP-Adapter） |
| **体型一致性** | ⭐ 低（只控制面部，不控制体型） |
| **发型一致性** | ⭐⭐ 中-低（取决于面部 embedding 是否含发型信息） |
| **服装一致性** | ❌ 无 |
| **适用场景** | 面部特写镜头、需要同一张脸的多个情绪/角度 |
| **参数建议** | FaceID weight: 0.8-1.0 |
| **诚实评级** | ⭐⭐⭐⭐ 面部专用。搭配 IP-Adapter 使用效果最佳 |

### 策略 4：LoRA 定制训练

| 维度 | 评估 |
|------|------|
| **原理** | 用 10-30 张该角色的图片训练一个轻量级 LoRA 模型 |
| **成本** | 中（需要 10-30 张训练图 + 30min-2h 训练时间） |
| **面部一致性** | ⭐⭐⭐⭐ 高（如果训练数据充足） |
| **体型一致性** | ⭐⭐⭐ 中-高 |
| **发型一致性** | ⭐⭐⭐⭐ 高 |
| **服装一致性** | ⭐⭐⭐⭐ 高（如果训练数据包含了该服装的多个角度） |
| **适用场景** | 需要大量生成的长期项目（如系列广告、多集短片） |
| **缺点** | 需要先有角色的多角度图片（鸡生蛋问题）、训练需要时间、换装需重新训练 |
| **诚实评级** | ⭐⭐⭐⭐⭐ 最佳质量。成本最高，适合角色出镜 5+ 次的严肃项目 |

### 策略 5：参考图重投（Reference Reroll）

| 维度 | 评估 |
|------|------|
| **原理** | 不用任何技术手段，手选看起来像同一角色的生成结果 |
| **成本** | 低（但需要大量生成 + 人工挑选） |
| **面部一致性** | ⭐⭐ 中-低（取决于运气和生成量） |
| **体型一致性** | ⭐⭐ 中-低 |
| **发型一致性** | ⭐⭐⭐ 中（发型是相对容易一致的特征） |
| **服装一致性** | ⭐⭐⭐ 中（服装 prompt 精确定义时一致性较好） |
| **适用场景** | 快速原型、低预算项目、仅需 2-3 张图且对一致性要求不高的场景 |
| **诚实评级** | ⭐⭐ 可用但不推荐。适合「差不多就行」的场景 |

### 策略 6：ControlNet + 线稿引导

| 维度 | 评估 |
|------|------|
| **原理** | 用同一张角色线稿/深度图作为 ControlNet 输入，约束生成结果的轮廓和姿态 |
| **成本** | 低（需要线稿/深度图） |
| **面部一致性** | ⭐⭐ 中-低（ControlNet 约束轮廓，不约束面部细节） |
| **体型一致性** | ⭐⭐⭐⭐ 高（轮廓约束非常有效） |
| **发型一致性** | ⭐⭐ 中 |
| **服装一致性** | ⭐⭐⭐ 中-高（如果服装轮廓固定） |
| **适用场景** | 固定姿势的多个角度/灯光变化 |
| **诚实评级** | ⭐⭐⭐ 姿势/轮廓专用。与 IP-Adapter 配合使用效果好 |

## 推荐组合策略

### 最优组合（推荐给用户）

```
严肃项目 → LoRA（如有资源）+ IP-Adapter（跨镜头）+ FaceID（面部特写）
一般项目 → IP-Adapter + 精确 prompt + 参考图重投（挑图）
快速原型 → 精确 prompt + 参考图重投
```

### 组合使用示例

```
1. 先在 Phase 3 的 Art Director 阶段生成 1 张角色定妆照（用 Flux，最佳质量）
2. 以这张定妆照作为 IP-Adapter 的参考图
3. 在 Phase 6 分镜阶段，每个含该角色的分镜 panel 都使用 IP-Adapter + 同一张参考图
4. 如果有面部特写需求，追加 FaceID 模块
5. 生成后人工挑选一致性最好的图——不要假设第一张就是最好的
```

## 诚实标注模板

> Agent 在 Phase 3 角色设计完成后，必须向用户展示以下诚实话术。

```markdown
### 角色一致性说明

已为角色「[角色名]」生成定妆照。此角色将在后续分镜中出现 [N] 次。

⚠️ **一致性限制（请知悉）**：
- AI 生图中跨镜头保持角色 100% 一致是目前尚未完全解决的技术难题
- 在本次项目中可达的一致性水平：约 70-80%（使用 [IP-Adapter / LoRA / 精确 prompt]）
- 面部特征的一致性 ≈ [高/中/低]，服装的一致性 ≈ [高/中/低]

**已采取的策略**：
- [策略 1 + 参数]
- [策略 2 + 参数]

**建议**：
- 如果需要更高的一致性，可以考虑训练角色 LoRA（约需 10-30 张定妆照 + 1-2h 训练）
- 或者在最终剪辑中选择角色出镜最一致的镜头使用
```
