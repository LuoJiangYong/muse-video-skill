# 产品演示 — Product Demo

> **场景特征**：产品是唯一的明星。每个镜头都在回答「这个产品能做什么」「为什么你需要它」。叙事服务于功能展示，功能展示服务于购买决策。
> **角色激活优先级**：Writer（功能→叙事映射）> DP（产品最佳角度）> VFX（功能可视化特效）> Director（信息清晰度审核）。

---

## 场景特征

| 维度 | 描述 |
|------|------|
| **空间** | 产品展示台 + 使用场景（棚内 + 少量外景） |
| **时间** | 15-120s / 每个功能展示 3-8s |
| **核心目标** | 让观众理解产品功能 → 产生购买/使用欲望 |
| **信息载体** | 产品画面 60% + 功能标注/UI 20% + 旁白 20% |

### 关键差异化参数（vs 其他场景）

| 参数 | 产品演示独有 | 其他场景对比 |
|------|------------|------------|
| **产品是唯一焦点** | 产品占画面 40-60%，背景和人都是辅助 | studio-ad 可用演员/场景做焦点，sci-fi 焦点在环境和氛围 |
| **功能→镜头映射** | 每个功能有明确的镜头方案（一对一映射） | 其他场景以情绪/叙事为组织逻辑 |
| **B-roll 配比公式** | 有精确的 A-roll/B-roll 配比 | studio-ad 全是 A-roll，sci-fi 无此概念 |
| **CTA 时机** | 必须设计 CTA 的落点 | studio-ad 有 CTA 但不强制每片都有 |
| **UI/标注叠加** | 大量使用（功能名/箭头/数据/对比） | 其他场景几乎不用 UI 标注 |

## 功能→镜头映射

### 映射原则

```
每个产品功能 = 1-2 个专用镜头

功能类型 → 镜头策略：
├─ 外观/设计 → Hero Shot（45° + 正面）+ 材质特写（微距）
├─ 操作/交互 → 使用场景（中景，手操作产品）+ UI 屏幕特写
├─ 效果/结果 → 前后对比（分屏或溶解转场）
├─ 尺寸/便携 → 手持展示 + 环境参考物对比
├─ 速度/性能 → 动态跟踪 + UI 计时器叠加
└─ 材质/工艺 → 微距旋转展示 + 交叉偏振光去反射
```

```json
{
  "feature_shot_mapping": [
    {
      "feature": "功能名称",
```

## B-roll 配比公式

### A-roll vs B-roll 定义

| 类型 | 定义 | 产品演示中的内容 |
|------|------|----------------|
| **A-roll** | 主线内容——产品在画面中 | 产品展示、功能操作、结果呈现 |
| **B-roll** | 辅助内容——建立氛围和上下文 | 使用环境、人物反应、生活方式片段、细节装饰 |

### 配比公式

```
15s 产品演示：  A-roll 85% / B-roll 15%（几乎全是产品）
30s 产品演示：  A-roll 70% / B-roll 30%
60s 产品演示：  A-roll 60% / B-roll 40%
90s+ 产品演示： A-roll 50% / B-roll 50%

B-roll 插入位置：
  - 功能段之间（作为「呼吸点」）
  - 功能对比的「使用前」状态
  - 结尾 CTA 前的情感铺垫
```

## CTA（行动号召）时机

### CTA 位置决策

```
片长 → CTA 位置和数量：

15s → 1 个 CTA，结尾 12-15s
30s → 1 个 CTA，结尾 25-30s；可选：开头 0-3s 预告 CTA
60s → 2 个 CTA：中间 25-30s（软 CTA）+ 结尾 55-60s（硬 CTA）
```

### CTA 类型

| CTA 类型 | 视觉形式 | 适用产品 |
|---------|---------|---------|
| **硬 CTA** | LOGO + 产品 + 文案（「立即购买」「了解更多」）+ URL/QR | 所有产品 |
| **软 CTA** | 产品 + 一句话 tagline（无直接购买指令） | 品牌建设、高端产品 |
| **预告 CTA** | 片头 2-3s 快速闪过一个悬念（「接下来 30 秒将改变你的 ___」） | 长演示（60s+） |

## 叙事模板

### 模板 1：问题-解决型（Problem-Solution）

```
结构：问题(20%) → 放大(20%) → 产品介入(30%) → 解决展示(20%) → CTA(10%)

0-20%：展示「没有这个产品时」的痛苦（B-roll 为主）
20-40%：问题放大——不只是不方便，而是「如果不解决会怎样」
40-70%：产品出现 → 功能逐个展示（A-roll 密集段）
70-90%：问题被解决的「美好结果」（B-roll 辅助）
90-100%：LOGO + CTA

案例引用：APPLE-DB（产品密度的功能叙事）
```

### 模板 2：功能巡礼型（Feature Tour）

```
结构：HOOK(10%) → F1(20%) → F2(20%) → F3(20%) → F4(20%) → CTA(10%)

0-10%：产品亮相（Hero Shot）
10-30%：功能 1 展示（核心卖点）
30-50%：功能 2 展示（差异化卖点）
50-70%：功能 3 展示（附加价值）
70-90%：功能 4 展示（情感连接）
90-100%：LOGO + CTA

注意：功能优先级排序——最打动人的功能放第一位（不是最重要的，是最打动人的）
```

### 模板 3：一天生活型（Day in the Life）

```
结构：早晨(20%) → 白天(40%) → 傍晚(20%) → 夜晚(20%)

0-20%：早晨场景中产品的第一次使用
20-60%：白天多个场景中产品的持续使用（蒙太奇）
60-80%：傍晚——产品帮助用户完成一件「额外的事」
80-100%：夜晚收尾 + 品牌落版

适用：随身/日用/生活方式类产品
```

## 角色参数

> 以下参数为各角色在产品演示场景下的默认配置。Agent 在激活对应角色时，需同时参考此处的场景特定参数。

### 灯光策略

| 参数 | 值 |
|------|-----|
| 主光源策略 | 多角度小光源，产品每个面都被定义 |
| 反差比 | 1:2-1:3 |
| 色温 | 5600K + 产品色 accent 灯 |

### 情绪曲线

| 参数 | 值 |
|------|-----|
| 曲线形状 | V 型（问题→解决） |
| 关键特征 | 0-30% 下坡，30% 拐点陡升，70-100% 持续高位 |

### 音效密度

| 参数 | 值 |
|------|-----|
| Foley 密度 | 高（每个功能操作） |
| Ambience 重要性 | 低 |
| Design SFX 用量 | 中（功能 whoosh） |
| UI 音效 | 中-高 |

### 视觉策略

| 参数 | 值 |
|------|-----|
| 空间布局 | 中景产品居中或 1/3 处；前景禁止遮挡（产品功能必须完全可见）；背景极简——纯白/纯黑/微渐变，无分散注意力的元素 |
| 深度策略 | 微距浅景深——产品焦点面锐利，非焦点面柔和过渡；背景虚化最大化；产品自身结构作为深度线索（边缘高光/阴影面） |
| 核心道具类型 | 产品（绝对主角）> 操作手（仅作为尺度参考和交互展示）> 无其他道具 |
| 色彩策略 | 产品原色主导 + 中性基底；禁止 accent 色抢产品色；全局色相偏差 ≤15°（极严格） |
| 材质方向 | 微距级——纹理/接缝/倒角/屏幕像素必须可见；材质对比用于功能分区（玻璃屏 vs 金属框） |
| 视觉重心优先级 | ① 当前展示的功能点 → ② 产品整体形态 → ③ 品牌 LOGO |
| 产品占比规则 | Hero Shot ≥50% 画面；功能特写 ≥70% 画面；品牌落版 30-40% |

## 常见陷阱

| 陷阱 | 避免方法 |
|------|---------|
| 功能列表式堆砌无节奏 | 用功能→镜头映射表，每个功能配独立镜头方案 |
| A-roll 100%，无呼吸点 | 功能段之间插 B-roll（至少 15-30% 配比） |
| CTA 突兀或缺失 | 按 CTA 位置决策表设计，硬 CTA 必须有 |
| 产品角度单一 | 每个功能至少换一个角度（正面/45°/顶视/微距） |
| 功能展示太快看不清 | 每个功能 3-8s，操作类功能加慢动作 |
