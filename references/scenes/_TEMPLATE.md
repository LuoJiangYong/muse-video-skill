# _TEMPLATE.md — 场景文档模板

> 使用说明：复制此文件为 `references/scenes/<scene-id>.md`，按模板填写。
> 场景 ID 必须与 SKILL.md 路由树中定义的值一致。运行 `python scripts/build_index.py --list-scenes` 查看当前可用值。
>
> **写作原则**：
> - 场景特征要具体可量化（「灯光可控度 100%」比「灯光很好」有效 100 倍）
> - 每个参数给出具体数值范围（色温用 K 值、反差比用比例、时长用秒数）
> - 角色激活优先级按实际重要度排序，不要写「全部同等重要」
> - 提示词模板要包含可直接复制到 image_gen 的格式，带示例
> - 常见陷阱要来自实战踩坑，不是「注意质量」这种废话

## 角色参数

> 以下参数为各角色在此场景类型下的默认配置。Agent 在激活对应角色时，需同时参考此处的场景特定参数。

### 灯光策略

| 参数 | 值 |
|------|-----|
| 主光源策略 | 【如：大柔光源 / 多角度小光源 / 底光为主 / 单一强背光】 |
| 反差比 | 【如：1:2-1:4 / 1:8-1:16 / 1:1 全亮】 |
| 色温 | 【如：5600K 日光 / 3200K 暖调 / RGB 自由 / 混合色温】 |

### 情绪曲线

| 参数 | 值 |
|------|-----|
| 曲线形状 | 【如：慢起→渐强→峰终 / V型 / 持续上升 / 波浪型】 |
| 关键特征 | 【如：0-40% 铺垫，最后 20% 直冲品牌 / 0-30% 下坡，30% 拐点陡升】 |

### 音效密度

| 参数 | 值 |
|------|-----|
| Foley 密度 | 【无 / 低 / 中 / 高 — 加具体说明】 |
| Ambience 重要性 | 【无 / 低 / 中 / 高 / 极高 — 加具体说明】 |
| Design SFX 用量 | 【无 / 低 / 中 / 高 — 加具体说明】 |
| UI 音效 | 【无 / 低 / 中 / 高 — 加具体说明】 |

### 视觉策略

| 参数 | 值 |
|------|-----|
| 空间布局 | 【前景/中景/远景各有什么，如：中景产品为主，前景禁止遮挡，背景极简纯色】 |
| 深度策略 | 【如何建立空间深度，如：浅景深分层 / 大气透视 / 尺度对比 / 2.5D 无传统深度】 |
| 核心道具类型 | 【什么道具定义了此场景的视觉身份，按优先级排列】 |
| 色彩策略 | 【主色方向 + 辅色策略 + 色相偏差容限，如：品牌色+中性基底，偏差≤20°】 |
| 材质方向 | 【主要材质类型和方向，如：产品材质必须清晰 / 金属拉丝+液态金属】 |
| 视觉重心优先级 | 【① → ② → ③ 及每个的占比规则】 |
| 空间情绪 | 【受控/干净 vs 混沌/有机 vs 巨物/压迫——空间本身传达的情绪】 |

> **Agent 使用说明**：Art Director 在 Phase 3 加载 §视觉策略；DP 在 Phase 4 加载 §灯光策略；Writer 在 Phase 2 加载 §情绪曲线；Sound Designer 在 Phase 5 加载 §音效密度。
> 各 role doc 的 Agent Prompt 已包含「加载 scene doc 获取场景特定参数」指令。
