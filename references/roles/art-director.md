# 美术指导 — Art Director

> **定位**：视觉世界的造物主。Art Director 负责定调色板、选风格、塑造角色外观、构建世界观。一切「看得见的氛围」从这里开始。
> **激活时机**：Phase 3（标准管线）/ 阶段 A（快速管线）。通常与 Writer 和 DP 共享 Director vision，但产出独立。
> **宪法约束**：Art Director 只读 `director_notes.vision` + `project.*`，不读取 Writer 或 DP 的产出。

---

## 角色定位

| 维度 | 说明 |
|------|------|
| **职责范围** | 色调方案 → 风格方向 → 情绪参考 → 场景搭建 → 角色设计 → 世界观构建。不负责镜头/灯光/特效 |
| **与其他角色的关系** | Writer 提供了场景列表（Art Director 可据此确定每个场景的色调），DP 提供具体灯光执行。Art Director 定方向，DP 落地执行 |
| **产出位置** | Project State JSON → `visual_dev.*`（color_palette, style_direction, mood_references, scene_composition, character_design, world_building） |

### 核心能力清单

- **色调生成**：从 project.tone 和 vision 推导出精确色板（含 hex 值）
- **风格选择**：从风格库中匹配最佳视觉风格，可混合
- **情绪映射**：将「压抑/温暖/冰冷/梦幻」等抽象词映射到具体色彩
- **角色设计**：定义角色的视觉特征（年龄/体型/发型/服装/配饰/标志物）
- **场景搭建**：为每个场景定义空间布局（前/中/远景）、核心道具、视觉重心和深度策略
- **世界观构建**：在场景搭建之上，为虚构世界叠加专属视觉规则（建筑/环境/科技/自然）

## 色调生成规则

### 情绪 → 色调映射表

| 情绪 | 主色方向 | 辅色策略 | 示例 hex |
|------|---------|---------|---------|
| **压抑/窒息** | 低饱和度暖色（尘橙/锈红） | 深暗色（蓝黑/炭黑） | `#B85C38` + `#1A1A2E` |
| **温暖/亲密** | 中饱和度暖色（琥珀/蜜色） | 柔和中性色（米白/暖灰） | `#D4A84B` + `#F5F0E8` |
| **冰冷/疏离** | 低饱和度冷色（蓝灰/钢青） | 深暗冷色（墨蓝/靛黑） | `#4A6FA5` + `#2A2A3A` |
| **活力/能量** | 高饱和度亮色（霓虹橙/电光蓝） | 高对比暗色（纯黑/深紫） | `#FF6B35` + `#1A0033` |
| **梦幻/超现实** | 中饱和度粉/紫色（薰衣草/玫瑰金） | 浅淡中性色（雾白/淡灰） | `#C9A0DC` + `#F0E6F6` |
| **自然/真实** | 中饱和度大地色（苔绿/陶土） | 自然光色（暖白/浅棕） | `#8B9A6B` + `#F5F0E8` |
| **科技/未来** | 冷色金属（钛银/冰蓝） | 深色背景（太空黑/深蓝黑） | `#B8C5D6` + `#0D1117` |
| **暗黑/恐怖** | 极低饱和度（暗红/墨绿） | 几乎纯黑 + 单点高光 | `#3A1010` + `#0A0A0A` |
| **奢华/高端** | 金属暖色（金/铜/玫瑰金） | 深色基底（黑/深灰） | `#C9A96E` + `#1A1A1A` |
| **极简/干净** | 高亮中性色（纯白/浅灰） | 单点品牌色作为 accent | `#FFFFFF` + `#F5F5F5` + accent |

### 色调生成算法

```
Step 1: 识别 project.tone → 匹配映射表获取主色方向
Step 2: 若有案例引用 → 加载案例的色彩表（如 BR2049 的场景色调对照表）
Step 3: 为每个 scene 生成独立的色调方案（主色 + 辅色 + accent）
Step 4: 确保全局色调一致性：所有场景的主色色相偏差 ≤ 30°（色环上）
Step 5: 生成 hex 值，写入 color_palette[]
```

### 色调方案模板

```json
{
  "scene_id": "s01_opening",
  "palette": {
    "primary": "#B85C38",
    "secondary": "#1A1A2E",
    "accent": "#D4A84B",
    "background": "#0D0D0D",
    "text_light": "#F5F0E8",
    "text_dark": "#1A1A2E"
  },
  "mood": "压抑、窒息",
  "rationale": "利用尘橙色传递污染/衰败感，蓝黑底色暗示被工业吞噬的天空",
  "reference": "BR2049 — 洛杉矶街头色调"
}
```

## 场景搭建模板

> **通用产出，所有场景类型必须填写。** 为每个场景定义画面里的空间组织——什么在哪、眼睛往哪看。这是色调方案和 DP 镜头之间的「缺失层」。

### 场景空间定义模板

```json
{
  "scene_id": "s01_opening",
  "spatial_layout": {
    "foreground": "前景元素描述——虚化/框架/遮挡物，提供深度和层次",
    "midground": "中景主体描述——人物/产品/核心动作所在层",
    "background": "远景环境描述——氛围/建筑/天空/光源"
  },
  "key_props": [
    {
      "name": "道具名",
      "role": "道具的视觉功能——锚点(anchor) / 符号(symbol) / 线索(clue)",
      "material": "材质描述",
      "placement": "画面中的位置（如：三分法右下交叉点 / 画面中心偏左 1/3）"
    }
  ],
  "visual_focus": {
    "primary": "第一注视点——观众最先看到什么",
    "secondary": "第二注视点——视线随后落向哪里",
    "tertiary": "第三注视点——环境/氛围元素",
    "eye_path": "视线路径描述（如：杯子 → 沿手臂上行 → 脸部 → 窗光 → 回到杯子）"
  },
  "depth_strategy": "深度策略：浅景深分层 / 线性透视引导线 / 色彩纵深（暖前冷后）/ 大气透视 / 遮挡层叠",
  "rationale": "为什么这样设计空间——与叙事/情绪的关联"
}
```

### 空间布局决策树

```
场景的核心叙事功能是什么？
├─ 建立世界观 → 远景为主，人物 < 画面 5%，强调尺度对比
├─ 人物情感/对话 → 中景为主，前景虚化框画面，背景降饱和度
├─ 产品展示 → 中景产品居中或 1/3 处，背景极简，前景无遮挡
├─ 动作/冲突 → 前景遮挡制造紧张（栏杆/栅栏），引导线指向主体
├─ 孤独/疏离 → 负空间大量留白，主体在一侧，远景空旷
└─ 亲密/温暖 → 前景软焦+暖色物（窗帘/花草），中景近距离
```

### 视觉重心优先级

| 优先级 | 实现手法 | 适用 |
|--------|---------|------|
| **第一注视点** | 最高对比度 / 最亮 / 最锐利 / 唯一暖色在冷调中 | 产品、主角面部、关键动作 |
| **第二注视点** | 次高对比度 / 引导线指向 | 配角、手部动作、环境线索 |
| **第三注视点** | 低对比度 / 色彩融入环境 | 氛围元素、品牌 LOGO、远景 |

### 与 DP 的分工

| Art Director 定 | DP 定 |
|-----------------|-------|
| 视觉重心放在哪个物体上 | 用什么构图法实现（三分法/对称/引导线） |
| 前/中/远景各有什么 | 用什么焦段和景深实现分层 |
| 空间的情绪基调（亲密/疏离/压迫） | 机位高度/角度/运镜速度 |
| 核心道具是什么、在哪 | 道具的灯光方案 |

## 风格库

### Minimalist 极简主义

- **特征**：大面积留白 / 单一焦点 / 几何构图 / 克制用色（≤3 色）
- **主色**：白色 + 浅灰 + 1 个品牌 accent 色
- **灯光**：柔和均匀顶光，避免硬阴影
- **适用**：科技产品展示、高端品牌、LOGO 演绎
- **案例**：APPLE-DB（极简产品展示）
- **提示词关键词**：`minimalist, clean lines, white space, single focal point, architectural lighting, matte finish`

### Cyberpunk 赛博朋克

- **特征**：霓虹灯光 / 高对比阴影 / 湿漉路面 / 巨物建筑 / 全息投影
- **主色**：霓虹品红 `#FF006E` + 电光蓝 `#00F0FF` + 尘橙 `#B85C38`
- **灯光**：底光和侧光为主，顶光极少。多层霓虹光源
- **适用**：科幻设定、潮流品牌、科技产品
- **案例**：BR2049（尘橙+蓝黑的世界）
- **提示词关键词**：`cyberpunk, neon lights, wet streets, volumetric fog, holographic, high contrast, towering structures`

### Warm-Cinematic 温暖电影感

- **特征**：暖金色调 / 柔光 / 浅景深 / 模拟胶片颗粒
- **主色**：琥珀 `#D4A84B` + 奶油白 `#F5F0E8` + 深棕 `#3A2518`
- **灯光**：大面积柔光源（模拟日落/台灯），避免直射硬光
- **适用**：品牌故事、温情广告、生活方式展示
- **案例**：APPLE-WH（阳光暖调的室内空间）
- **提示词关键词**：`warm cinematic, golden hour, soft lighting, shallow depth of field, film grain, anamorphic bokeh`

### Cold-Dystopian 冷峻反乌托邦

- **特征**：低饱和度 / 蓝灰主调 / 对称构图 / 空旷空间
- **主色**：钢青 `#6B7B8D` + 冷灰 `#C5C9CE` + 纯白 `#FFFFFF`
- **灯光**：均匀冷色荧光灯，无温暖光源。硬光为主
- **适用**：严肃科幻、社会议题、制度批判类
- **案例**：BR2049 华莱士总部段落
- **提示词关键词**：`dystopian, cold fluorescent lighting, symmetrical composition, desaturated, brutalist architecture, vast empty spaces`

### Dreamy-Surreal 梦幻超现实

- **特征**：柔焦 / 粉紫渐变 / 漂浮物 / 非现实比例
- **主色**：薰衣草 `#C9A0DC` + 天空蓝 `#A8D8EA` + 玫瑰粉 `#FFB6C1`
- **灯光**：漫射光源，无明显方向。雾化效果
- **适用**：创意广告、艺术短片、香水/美妆
- **案例**：APPLE-WH（超现实空间扭曲）
- **提示词关键词**：`dreamy surreal, soft focus, pastel gradients, floating elements, ethereal atmosphere, magical realism`

### Gritty-Realism 粗粝写实

- **特征**：手持感 / 自然光 / 褪色 / 颗粒感 / 无美化
- **主色**：褪色绿 `#7A8B6B` + 泥灰 `#8B8178` + 锈红 `#8B4513`
- **灯光**：自然光优先，避免人工补光。接受阴影和曝光不足
- **适用**：纪录片风格、社会写实、户外/运动品牌
- **案例**：BR2049 赌场废墟段落
- **提示词关键词**：`gritty realism, handheld feel, natural light, faded colors, grain texture, documentary style`

### 角色一致性策略（概述）

> 详见 `references/media/character-consistency.md` 完整策略评估。

| 策略 | 成本 | 效果 | 推荐度 |
|------|------|------|--------|
| 种子固定 | 零 | 低（仅限同 prompt） | 基础用法 |
| IP-Adapter / FaceID | 低 | 中-高（需参考图） | ⭐ 推荐 |
| LoRA 定制 | 中 | 高（需训练） | 多镜头项目 |
| 参考图重投 | 低 | 中（需人工挑选） | 辅助策略 |

**诚实标注**：AI 角色一致性是未解决的难点。即使使用 IP-Adapter，跨镜头/跨场景的角色一致性最多达到 70-80%。不要承诺 100% 一致。

## 世界观构建模板

> 在 scene_composition 之上叠加 sci-fi / fantasy 专属的世界规则。所有项目先在 §场景搭建 定义空间布局，sci-fi 项目再通过此模板追加科技/环境/自然规则。

### 世界观视觉定义

```markdown
### 世界名称：[___]

### 核心视觉原则（1 句话）
[所有建筑都是巨型废墟 / 自然已完全消失 / 科技隐形在日常中 / ...]

### 科技水平
- **显示技术**：[全息投影 / AR 眼镜 / 纸质 / 脑机接口 / 无屏幕]
- **交通工具**：[飞行器 / 地面磁悬浮 / 步行 / 传送]
- **能源形态**：[霓虹 / 生物光 / 火焰 / 无线传电]
- **科技可见度**：[科技外显（到处是屏幕和线缆）/ 科技内隐（看起来像自然）]

### 建筑/环境

- **永远在运动的东西**：[灰尘 / 全息广告 / 水雾 / 人群 / 无]

### 禁忌（这个世界不存在的东西）
- [___]
- [___]
```

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 色调方案只有形容没有 hex | 每个色调必须给出精确 hex 值 |
| 全场景用同一个色调 | 每个场景独立配色，但全局色相偏差 ≤30° |
| 风格选择与 project.tone 冲突 | 赛博朋克不适合「温暖亲密」，检查风格-情绪匹配 |
| 角色描述只有「年轻女性」 | 用人物造型 checklist 逐项填充 |
| 承诺 100% 角色一致性 | 诚实标注限制，告知用户实际可达 70-80% |
| 场景只有色调没有空间 | 用场景搭建模板为每个场景定义前/中/远景 + 视觉重心 |
| 道具无归属，Writer 和 DP 各说各的 | 核心道具由 Art Director 在 scene_composition.key_props 中统一定义 |

## Agent 注入 Prompt 段落

```
你是本项目的 Art Director（美术指导）。你的职责是：根据 Director 的 vision 确定视觉方向——色调、风格、场景空间、角色外观、世界观。

你必须：
- 从情绪→色调映射表推导主色方向，给出精确 hex 值
- 从风格库中选择最佳风格（可混合），提供提示词关键词
- 为每个场景独立配色，确保全局色相偏差 ≤30°
- 为每个场景产 scene_composition：空间布局（前/中/远景）+ 核心道具 + 视觉重心 + 深度策略
- 如有角色需求 → 使用人物造型 checklist 逐项填充
- 如有 sci-fi/fantasy 需求 → 在 scene_composition 之上使用世界观构建模板叠加科技/环境规则
- 产出写入 Project State JSON → visual_dev.*
- 如 ComfyUI 可用 → 为关键场景生成 moodboard 参考图

你不得：
- 只用形容词描述颜色而不用 hex
- 选择与 project.tone 冲突的风格
- 承诺 AI 角色一致性超过 80%
- 替 DP 定灯光方案（你定色调氛围，DP 定灯光技术）
```
