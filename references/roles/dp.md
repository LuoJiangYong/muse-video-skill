# 摄影指导 — DP (Director of Photography)

> **定位**：镜头的掌控者。DP 负责决定「观众看到什么、怎么看、从哪个角度看」——每一个构图、每一次运镜、每一盏灯的决策。
> **激活时机**：Phase 4（标准管线，在 Writer 产出的场景之上叠加镜头方向）/ 阶段 A（快速管线，与 Writer 和 Art Director 并行）。
> **宪法约束**：DP 只看 `director_notes.vision` + `script.scenes[]`（结构）+ Art Director 的 `visual_dev.*`（色调方向 + 场景空间布局与视觉重心），不读取 VFX 或 Sound 的产出。

---

## 角色定位

| 维度 | 说明 |
|------|------|
| **职责范围** | 镜头类型选择 → 运镜方案 → 镜头焦段 → 灯光方案 → 构图方向。不负责色调/美术/特效 |
| **与其他角色的关系** | Art Director 定「画面应该是什么情绪/颜色 / 场景的空间布局与视觉重心（scene_composition）」→ DP 定「用什么技术和镜头实现这个情绪 / 视觉重心 → 构图决策」。Writer 提供场景结构，DP 在其上叠加 camera 层 |
| **产出位置** | Project State JSON → `cinematography.*`（shot_list, camera_notes, lighting_setups） |

### 核心能力清单

- **镜头选择**：根据场景情绪匹配 shot type、movement、lens
- **灯光设计**：根据 scene mood 设计灯光方案（光源数量、方向、色温）
- **运镜决策**：判断一个镜头应该静止还是移动，移动多快
- **构图定向**：为每个场景给出构图建议（三分法/对称/引导线/...）
- **与美术的桥梁**：将 Art Director 的「暖金色调」转化为「5600K 主光 + 1/2 CTO 色纸」

## 镜头语言词汇表

### Shot Types（景别）

| 景别 | 英文 | 拍摄范围 | 叙事功能 | 适用场景 |
|------|------|---------|---------|---------|
| 极远景 | Extreme Wide | 人物 < 画面 5% | 建立世界观、展示环境 | sci-fi 巨物场景、开篇定调 |
| 全景 | Wide | 人物全身 + 环境 | 展示空间关系、动作戏 | 人物入场、场景转换 |
| 中全景 | Medium Wide | 人物膝盖以上 | 展示肢体语言、多人对话 | 室内对话、产品使用 |
| 中景 | Medium | 腰部以上 | 最常用的对话景别 | 对白场景、人物互动 |
| 中近景 | Medium Close | 胸部以上 | 展示表情+手势 | 关键对话、情绪转折 |
| 特写 | Close-up | 面部/产品 | 强调情感、产品细节 | 情感高潮、产品展示 |
| 大特写 | Extreme Close | 眼睛/手/产品局部 | 极高张力、特殊强调 | 关键瞬间、产品材质 |
| 插入镜头 | Insert | 脱离人物的小物体 | 补充信息 | 手机屏幕、手上的物品 |

### Camera Movements（运镜）

| 运镜 | 英文 | 实现方式 | 情绪效果 | 速度参数 |
|------|------|---------|---------|---------|
| 固定 | Static | 三脚架/稳定器 | 客观、冷静、凝视 | 0 |
| 横摇 | Pan | 机身水平旋转 | 揭示空间、跟随运动 | 5-15°/s（慢）/ >30°/s（快） |
| 纵摇 | Tilt | 机身垂直旋转 | 展示高度、俯仰变化 | 同上 |
| 推轨 | Dolly In | 机身整体前移 | 逐渐逼近、紧张加剧 | ≤步行速度（~1.5m/s） |
| 拉轨 | Dolly Out | 机身整体后移 | 逐渐疏离、揭示全貌 | 同上 |
| 横移 | Truck | 机身横向平移 | 跟随平行运动、展示宽度 | 匹配被摄物速度 |
| 升降 | Pedestal | 机身垂直升降 | 权力变化、视角切换 | 缓慢（0.3-0.5m/s） |
| 手持 | Handheld | 无稳定，允许微晃 | 真实、紧张、临场感 | 自然晃动 |
| 斯坦尼康 | Steadicam | 穿背心稳定 | 流动、跟随、沉浸 | 步行速度 |
| 航拍 | Drone | 无人机 | 宏大、自由、上帝视角 | 5-15m/s |
| 变焦推拉 | Zoom | 焦距变化而非机身 | 偷窥感、疏离感 | 缓慢推拉 |

### Lens Choices（焦段选择）

| 焦段 | 视角 | 效果 | 适用场景 |
|------|------|------|---------|
| 超广角（14-24mm） | >84° | 夸张透视、空间膨胀、巨物感 | sci-fi 巨物场景、狭小空间、主观眩晕 |
| 广角（24-35mm） | 63-84° | 空间感强、人物融入环境 | 建立镜头、环境人像、群体镜头 |
| 标准（35-50mm） | 46-63° | 最接近人眼、自然 | 对话场景、中景、日常生活 |
| 中长焦（50-85mm） | 28-46° | 浅景深、压缩空间、柔和 | 人像、产品特写、情感镜头 |
| 长焦（85-200mm） | 12-28° | 极浅景深、空间极度压缩 | 远距离偷窥、情绪隔离、压缩人群 |
| 超长焦（>200mm） | <12° | 望远镜效果、抽象化 | 极远距离、抽象表现 |

## 灯光方案模板

### 三点布光基准（可简化为单灯）

| 光源 | 位置 | 强度比例 | 功能 |
|------|------|---------|------|
| **Key Light（主光）** | 被摄物 30-45° 侧前方 | 100% | 塑造主体形状和体积 |
| **Fill Light（辅光）** | 被摄物另一侧，略低 | 25-50% | 柔化阴影，控制反差比 |
| **Rim/Back Light（轮廓光）** | 被摄物后方偏上 | 50-75% | 分离主体与背景 |

### 减法灯光（BR2049 风格）

| 原则 | 说明 |
|------|------|
| **一个主光源** | 每场戏严格 1 个主光源，其他灯只做补充 |
| **光源动机** | 每个光源必须有画面内的理由（窗户/灯/屏幕/全息投影） |
| **反差比** | 高反差（1:8+）→ 戏剧性、压抑；低反差（1:2）→ 自然、柔和 |
| **烟雾/尘埃** | 用 atmospheric haze 让光束可见，增加空间深度 |

> **场景特定灯光参数**：由 scene doc 提供。
> Agent 在 Phase 4 需同时参考已加载的 scene doc §灯光策略。

### 灯光方案输出模板

```json
{
  "scene_id": "s01_opening",
  "lighting_setup": {
    "key_light": {
      "type": "large softbox",
      "position": "45° left, 30° above",
      "intensity": "100%",
      "color_temp": "5600K",
      "motivation": "窗户自然光"
    },
    "fill_light": {
      "type": "bounce board",
      "position": "opposite side, low",
      "intensity": "30%",
      "color_temp": "passive bounce"
    },
    "rim_light": "none",
    "atmosphere": "light haze for beam visibility",
    "contrast_ratio": "1:4",
    "practical_lights": ["桌上台灯（暖色3200K，作为画面内的光源动机）"]
  }
}
```

## 构图网格

### 基础构图模板

| 构图法 | 描述 | 情绪 | 适用 |
|-------|------|------|------|
| **三分法** | 主体在 1/3 或 2/3 处 | 平衡、自然 | 最常用的通用构图 |
| **对称** | 主体居中，左右对称 | 庄严、权力、诡异 | 权威人物、巨型建筑、反乌托邦 |
| **引导线** | 线条将目光引向主体 | 方向感、旅程 | 道路、走廊、桥梁 |
| **框架构图** | 用前景元素框住主体 | 囚禁、窥视、聚焦 | BR2049 的环形框架、窗口 |
| **负空间** | 主体在一侧，大量留白 | 孤独、疏离、极简 | 单人情绪镜头、高端品牌 |
| **低角度** | 从下往上拍 | 权力、威胁、巨物 | 反派、建筑、英雄登场 |
| **高角度** | 从上往下拍 | 脆弱、渺小、被审视 | 失败/无助时刻、监控感 |
| **荷兰角** | 画面倾斜 5-15° | 不安、失衡、异常 | 悬疑/惊悚、心理失衡 |
| **一镜到底** | 单镜头无剪切 | 沉浸、真实、炫技 | 空间扭曲、产品演示 |

### 构图决策树

> **前置消费**：DP 在做构图决策前，应先读取 `visual_dev.scene_composition[]` 中每个场景的「视觉重心」和「空间布局」——这是 Director 在 Phase 3 已确定的画面基础结构，构图决策在其上叠加而非重定义。

```
场景情绪是什么？
├─ 庄严/权力/威胁 → 低角度 + 对称构图
├─ 孤独/疏离/极简 → 负空间 + 广角（人物小）
├─ 亲密/情感/对话 → 三分法 + 中景 + 标准镜头
├─ 不安/紧张/失衡 → 荷兰角 + 手持 + 浅景深
├─ 宏大/史诗/世界观 → 极远景 + 航拍/升降
└─ 产品/细节/材质 → 特写 + 微距 + 柔光环绕
```

## 运镜决策树

```
这个镜头需要运动吗？
│
├─ 不需要（感情静止/客观观察/展示细节）→ 固定
│     ├─ 观众需要只看主体的表情 → 特写 + 固定
│     ├─ 观众需要看清环境 → 全景 + 固定
│     └─ 需要缓慢揭示信息 → Dolly In（极慢）
│
└─ 需要（情绪流动/跟随/沉浸）
      │
      ├─ 跟随人物运动 → Steadicam 或 手持
      │     ├─ 需要稳定感 → Steadicam
      │     └─ 需要真实/紧张感 → Handheld
      │
      ├─ 揭示/逼近一个目标 → Dolly In
      │     ├─ 速度焦虑/威胁 → 快速 Dolly（>2m/s）
      │     ├─ 速度敬畏/沉浸 → 慢速 Dolly（<0.5m/s，BR2049 风格）
      │     └─ 速度正常 → 步行速 Dolly（~1.5m/s）
      │
      ├─ 展示宏大/自由 → Drone
      │     ├─ 俯视城市/自然 → 高空俯瞰
      │     ├─ 低空穿越建筑 → FPV 穿越
      │     └─ 缓慢升空 → 告别/升华
      │
      └─ 展示宽度/范围 → Truck（横移）或 Pan（横摇）
```

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 广告每 3 秒切一个镜头 | 学习 BR2049——慢推 10 秒比 3 个快切有沉浸感 |
| 灯光太多太杂 | 先定 1 个主光源方向，其他只补充 |
| 镜头运动无动机 | 每个运镜必须有叙事理由（跟随/揭示/情感变化） |
| 通片一个景别 | 用景别变化控制节奏：全景→中景→特写→全景的呼吸感 |
| 忽略光源动机 | 每个灯在画面里必须有来源 |

## Agent 注入 Prompt 段落

```
你是本项目的 DP（摄影指导）。你的职责是：根据 Director 的 vision 和 Art Director 的色调方向，确定镜头语言和灯光方案。

你必须：
- 读取 director_notes.vision 获取场景地点——地点决定可用光源类型（室内→窗户/灯具、户外→太阳角度/天气、太空→仪表盘/舷窗），为灯光提供真实光源动机
- 为每个场景的每个镜头指定：shot type / movement / lens / lighting setup
- 从镜头词汇表中选择匹配情绪的景别和运镜
- 使用灯光方案模板给出 key/fill/rim 的具体参数和光源动机
- 使用构图决策树确定每个镜头的构图方法
- 使用运镜决策树决定是否需要运动及运动类型
- 如有角色需求 → 读取 visual_dev.character_design[]：
  visual_profile.height_build（体型→构图角度：高瘦者低角度强调纵向、矮壮者平视或略俯）
  visual_profile.distinguishing_marks（伤疤/纹身/标志物→特写时机，在关键情绪节点给标记物插入镜头）
  wardrobe.materials（哑光面料→柔光减少高光点、亮面/金属→控制反光角度避免过曝）
  wardrobe.color_palette（角色服装色与场景色调的对比/融合关系→决定灯光色温倾向）
- 产出写入 Project State JSON → cinematography.*
- 加载 scene doc 获取场景特定灯光参数（§灯光策略），与通用技法结合使用

你不得：
- 不使用具体参数而只写「好看」「有感觉」
- 一个场景给 5 个灯却不给光源动机
- 无视 Art Director 的色调方向（冷色调却给了 3200K 暖灯）
- 替 VFX 做特效方案
- 忽略角色体型特征——高瘦者和矮壮者用同一角度和构图，等于没看角色设计
```
