## 叙事技法（→ Writer）

### 功能叙事：每个功能 1 句话

- **表现**：旁白以极快的语速逐一列举 Apple 2019 年发布的新产品功能——每个功能配 1 个画面，每个画面约 0.5-1.5 秒。全片约 80+ 个功能点在 107 秒内完成。功能之间没有过渡段落、没有情感铺垫——功能 A → 功能 B → 功能 C，直线前进。
- **手法**：「列举式叙事」——不是讲故事，而是做展示。每个功能 = 1 句话旁白 + 1 个精准画面。观众不需要理解每个功能如何运作，只需要感知「Apple 做了很多很多事」。
- **可迁移**：如果是多产品/多功能展示，使用列举式结构。公式：
  - 每个功能 = 画面 1-2s + 旁白 1 句（≤ 8 词）
  - 功能之间的间隔 = 0（硬切）
  - 总时长 = 功能数 × 1.5s + 开头 5s（标题）+ 结尾 5s（品牌落地）
  - 107s 约容纳 65-80 个功能点
- **适用场景**：product-demo

### 极简旁白：每句话都是子弹

- **表现**：旁白无主语、无连接词、无修饰语。「Retina display. All-day battery. Face ID. Water resistant.」——每个词都是产品功能，每个词后面跟着一个证明它的画面。全片没有一句废话。
- **手法**：旁白写作的「子弹规则」：
  - 每句 ≤ 6 个词
  - 每句只描述 1 个功能
  - 不需要完整句子——名词短语即可（"A13 Bionic. The fastest chip in a smartphone."）
  - 禁止连接词：没有 "and", "also", "plus", "with"
  - 禁止情感词汇：没有 "amazing", "incredible", "revolutionary"——让画面自己说
- **可迁移**：写产品功能的旁白时，先列出所有功能，然后把每个功能的描述压缩到 6 词以内。删掉所有形容词、连接词、过渡句。剩下的就是你的旁白。
- **适用场景**：product-demo

### 密度递增：从慢到快的信息释放

- **表现**：
  - 开头（0:00-0:10）：缓慢——3 个产品依次浮现，每个约 3 秒
  - 中段（0:10-1:00）：加速——功能开始快速切换，约 1-1.5s/功能
  - 高潮（1:00-1:35）：极速——约 0.5-1s/功能，画面几乎在闪烁
  - 结尾（1:35-1:47）：骤停——最后 12 秒回到慢速，一个产品/一个功能/一个画面
- **手法**：密度曲线 = 开始时让观众「跟上」→ 中间让观众「被淹没」→ 结尾让观众「被震撼」。关键：「被淹没」不是失误——它是设计的一部分。观众不需要在中间段记住每个功能，只需要感受「太多了」。
- **可迁移**：设计信息密度的「呼吸」——不能让观众全程高能（会疲劳），也不能全程低速（会无聊）。S 曲线：慢启动 → 快速爬升 → 极速 → 骤停。骤停后的「余震」= 品牌印象最深的时刻。
- **适用场景**：product-demo, studio-ad

## 镜头语言（→ DP）

### 高速剪辑：0.5 秒的构图书

- **表现**：峰值段每个镜头持续 0.5-1.0 秒。在这个时间内需要完成：产品识别、功能理解、美感感知。这意味着每个镜头在出现的第一帧就必须已经「在构图上完整」——没有推拉摇移的余地。
- **技术参数**：
  - 景别：90% 为产品特写（产品占画面 60-80%）
  - 焦点：产品永远在焦点上（f/5.6-f/8 确保足够景深）
  - 构图：产品居中或 1/3 处——不用创造性构图（观众没时间欣赏）
  - 运动：90% 静态机位 + 产品旋转/滑动入画，而非摄影机运动
  - 每镜第一帧 = 最终构图（不留给构图调整的时间）
- **可迁移**：做高速剪辑的构图金律——「第一帧即最终帧」。每个镜头在编辑时间线上必须是：第一帧就能看懂，不需要等待。测试方法：在时间线上只播放每个镜头的第一帧——如果第一帧你看不懂这个镜头在说什么，它就不适用于高速剪辑。
- **适用场景**：product-demo

### 产品密度的空间设计

- **表现**：画面中的每个元素都是产品的一部分——没有无关的背景、人物、环境。暗色产品台上，产品悬浮/旋转/滑动，背景为纯黑或极暗灰。任何光线都是为产品服务的。
- **技术参数**：
  - 背景：纯黑 `#000000` 或极暗灰 `#0D0D0D`（RGB 13, 13, 13）
  - 灯光：单一产品台灯光——顶光柔光箱 + 侧光（产品边缘高光）
  - 产品间距：画面中产品与边缘的距离 ≥ 产品的 20%（留出呼吸空间）
  - 反射控制：金属产品用交叉偏振光消除杂乱反射，玻璃产品用底部透光
- **可迁移**：产品展示的「减法原则」——从画面中去掉一切不是产品的东西。背景 = 纯黑。灯光 = 只为产品服务。运动 = 产品自己动，摄影机不动。
- **适用场景**：product-demo

### 微距凝视：功能的视觉化翻译

- **表现**：当旁白说「Ceramic Shield」时，画面不是 iPhone 放在桌上——而是微观层面的陶瓷晶体结构动画。当旁白说「Deep Fusion」时，画面是像素级的图像处理过程可视化。功能不是被「描述」的——是被「看到」的。
- **技术参数**：
  - 微距镜头（100mm 等效）+ 环形灯
  - 素材提升（CG 微观动画 + 实拍微距混合）
  - 焦段内产品细节放大到 1:1 或更高倍率
- **可迁移**：对于抽象功能（处理速度/材料强度/算法智能），不要用图表解释——做微距视觉化。让观众「看到」陶瓷晶体、「看到」像素处理过程。越抽象的功能越需要具象的视觉化。
- **适用场景**：product-demo

## 色彩/美术（→ Art Director）

| 场景/段落 | 主色 (hex) | 辅色 (hex) | 情绪 | 设计理由 |
|-----------|-----------|-----------|------|---------|
| 全片背景 | `#000000` 纯黑 | `#0D0D0D` 极暗灰 | 专注、高级、无干扰 | 纯黑背景 = 产品是唯一的光源。任何背景色都会稀释产品的视觉冲击力。 |
| 产品材质光 | `#F5F5F5` 冷白 | `#E0E0E0` 亮灰 | 精密、技术 | 产品高光统一为冷白（5500-6000K），不偏暖——传达「科技=精确」而非「科技=温暖」。 |
| UI 叠加色 | `#007AFF` Apple蓝 | `#34C759` Apple绿 | 品牌、功能 | UI 标注（功能名/箭头）使用 Apple 品牌色，但与产品颜色保持对比。 |
| 渐变过渡 | `#1A1A2E` 深蓝黑 | `#2D1B69` 暗紫 | 深度、科技 | 罕见的色彩时刻——产品之间的转场偶尔用一帧深蓝/紫色渐变，像「黑暗中的闪电」。 |
| 结尾 Apple Logo | `#FFFFFF` 纯白 | `#000000` 纯黑 | 标志性、权威 | 全片唯一纯白元素——Apple 的 LOGO 在纯黑背景上像太阳。107 秒的密集信息后，白 LOGO 让眼睛「休息」。 |

### 世界观/场景设计要点

- **「黑色无限空间」**：没有「场景」——只有产品。黑色不传达「黑暗」——它传达「无限」。产品漂浮在无限的可能性中。
- **光线 = 产品身份**：每个产品有专属的光线设计。iPhone 用侧光突出边框，MacBook 用顶光突出键盘，AirPods 用底光突出透明充电盒。光线定义了产品的「个性」。
- **无人物原则**：全片没有人的手或脸出现在画面中（仅有少量镜头显示手指操作屏幕）——这是刻意选择。人物的缺席 = 观众可以想象「我」在操作这些产品。

## 特效语言（→ VFX）

### UI 功能标注叠加

- **表现**：功能名称和简短说明以 Apple 风格的 UI 元素形式叠加在产品画面上——但字体极小、位置极边缘。功能标注不是为了让你「阅读」——它是为了让你「感知」到有很多功能被列出了。
- **实现思路**：
  - 字体：SF Pro Display, Light weight, 12-16px
  - 位置：画面底部或边缘，不遮挡产品主体
  - 动画：淡入（0.3s）+ 停留（0.5-1s）+ 淡出（0.2s）
  - 颜色：`#007AFF`（Apple蓝）或 `#8E8E93`（Apple灰）
- **可迁移**：
  - HyperFrames：用 CSS `@keyframes` + `opacity` 做 UI 标注的淡入/淡出。字体用系统默认 sans-serif 模拟 SF Pro 风格。
  - 交互视频：如果观众可以暂停，标注可以详细。如果无法暂停（线性播放），标注只给「印象」不给「信息」——观众需要暂停才能看清 = 失败。
- **适用场景**：product-demo

### 产品极速转场

- **表现**：产品 A → 产品 B 的转场在 1-2 帧内完成。常用手法：
  - 产品旋转（A 旋转出画 → B 旋转入画，2 帧）
  - 比例匹配转场（A 的某个圆形元素 → 下一帧 B 的圆形元素在同一位置）
  - 光闪过场（1 帧全白 → 下一帧新产品已就位）
- **实现思路**：
  - 旋转转场：A 的 Y 轴旋转 90°（2 帧内）→ B 从 Y 轴 -90° 旋转至 0°（2 帧内）。总转场 ≤ 4 帧（0.13s@30fps）。
  - 匹配转场：找到 A 和 B 共有的视觉元素（圆形 Home 键 / 摄像头 / 屏幕圆角），将两者缩放到相同尺寸/位置，做溶解转场（2 帧）。
- **可迁移**：
  - HyperFrames：用 CSS `transform: rotateY()` + `transition: 0.1s` 做快速旋转转场。
  - ComfyUI：产品之间的过渡用 `ImageBlend` + `dissolve` mode，混合帧数 = 1-2。
- **适用场景**：product-demo, logo-animation

## 声音设计（→ Sound Designer）

### 极简音景：让旁白成为音乐

- **表现**：音乐极简——低沉的合成器 Pad（持续长音）+ 稀疏的鼓点（每 2-4 小节一次）。音乐不抢旁白的注意力。旁白本身的「节奏感」（每句的断句、语气、速度）成为全片的「真正音乐」。
- **声音层次**：旁白 (55%) > 极简音乐 Pad (25%) > 产品 SFX (15%) > 鼓点 (5%)。旁白被推到前景——它是全片最响、最清晰的声音元素。
- **可迁移**：当旁白是核心信息载体时，音乐必须退让。降低音乐在 200-800Hz（人声频段）的音量——EQ 凹陷 3-5dB。音乐的作用不是「好听」——是「填充旁白之间的空隙」。
- **适用场景**：product-demo

### 节奏递增的音频设计

- **表现**：音频的节奏递增与视觉同步：
  - 开头：只有 Pad 长音（无鼓点，无 SFX）
  - 中段：鼓点每 4 小节一次 + 少量产品 SFX（旋转声/点击声）
  - 高潮：鼓点每 1 小节一次 + 密集产品 SFX（多个声音叠加）+ Pad 音量调高
  - 结尾：所有音乐骤停 → 只有旁白最后一句 → 然后是 2 秒的「耳中的余音」
- **可迁移**：音频节奏曲线的设计：用鼓点密度控制能量感。`鼓点密度 = 鼓点次数 / 时间`——从 0.1Hz 开始，线性增加到 2Hz（极值），骤降到 0。
- **适用场景**：product-demo

### 产品声音的符号化

- **表现**：每个产品交互有独特的音效——但这不是产品实际发出的声音。旋转声 = 低频 whoosh（非实际声音，但听起来像「厚重精密的东西在转动」）。点击声 = 高频短促 click（非实际触摸屏声音，但「感觉精准」）。
- **可迁移**：产品 SFX 不要用「实际声音」——用「理想化声音」。用低频传达「重量/品质」，用高频传达「精确/速度」。声音设计的目标不是真实——是「比真实更好的感知」。
- **适用场景**：product-demo

## 创意策略（→ 广告片专属）

### 产品密度 = 竞争力

- **表现**：107 秒展示 80+ 功能——观众不需要记住任何一个。他们只需要记住：「Apple 做了这么多东西」。产品密度本身成为品牌信息——「我们创新得如此之快，你眨眼就会错过」。
- **品牌目的**：在竞争对手发布 1-2 个产品时，Apple 展示了一整年的产品线。数量和速度 = 实力的证明。这是典型的「数量即质量」的广告策略。
- **可迁移**：如果你的产品线丰富/功能多样——做一个「数量展示」型广告。密集的信息释放让观众无法逐一评估 = 他们只能接受整体印象（「很多」→「强大」）。
- **适用场景**：product-demo

### 节奏递增的心理操控

- **表现**：广告从慢到快再到骤停的节奏设计，精确操控观众的注意力状态：
  - 慢速段：注意力聚焦（「我可以跟上」）
  - 加速段：注意力扩展（「等等，太快了」）
  - 极速段：注意力放弃（「我不需要跟上了，我只需要感受」）
  - 骤停段：注意力回收（「刚才发生了什么」）
- **品牌目的**：让观众从「理性评估产品」状态过渡到「感性接受品牌」状态。当观众放弃逐功能理解时，他们转而信任品牌——「Apple 的东西就是多/好/快」。
- **可迁移**：设计信息释放的「放弃拐点」——在某个点让信息的密度超过观众的处理能力，迫使他们从理性模式切换到感性模式。放弃拐点通常在总时长的 60-70% 处。
- **适用场景**：product-demo

### 功能叙事的「证明-不解释」

- **表现**：旁白说「Ceramic Shield」→ 画面展示微观陶瓷晶体。旁白说「Deep Fusion」→ 画面展示像素处理过程。旁白说「A13 Bionic」→ 画面展示芯片内部晶体管。每句话都被画面「证明」了——不需要解释这些词的意思。
- **品牌目的**：建立「Apple 说的话都是真的」的信任感。不是「请相信我们」——而是「我们给你看了证据」。
- **可迁移**：每当广告中有一个需要旁白解释的功能——问：「这个画面能证明这句话吗？」如果能，保留。如果不能——换一个能证明的画面，或者删掉这句话。
- **适用场景**：product-demo

## 适用提示

- **最佳匹配场景**：多产品/多功能展示、需要传达「数量=实力」的品牌广告、发布会开场/总结视频、产品线年度回顾。
- **不适用场景**：单产品深度展示（需要情感叙事）、需要建立角色/故事的广告、奢侈品/慢品牌（不需要「快」的印象）。
- **引用优先级**：核心引用（高速剪辑 / 产品密度 / 节奏递增 / 极简旁白 / 功能→画面映射）。
