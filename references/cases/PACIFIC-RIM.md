## 叙事技法（→ Writer）

### 双人共驾（Drift 系统）—— 天然的双主角关系

- **表现**：Jaeger 机甲必须由两个人同步操作——两人的大脑通过「Drift」系统神经连接。这意味着两个驾驶员不能有任何隐瞒（Drift 会共享记忆）、不能有任何迟疑（不同步=机甲瘫掉）、必须绝对信任对方。Raleigh 和 Mako 从「陌生人→拍档→战友→家人」的完整关系弧线，全部由 Drift 系统驱动。
- **手法**：Drift 不是「背景设定」，而是「叙事引擎」。(a) 每次 Drift 连接 = 角色关系的一次升级（从 Mako 第一次连接时的创伤闪回→最后决战时两人的完美同步）；(b) Drift 闪回 = 角色的过去能被对方「看到」= 省去所有闪回叙事的铺垫；(c) Drift 断开 = 死亡——这给每场战斗增加了一个「两个人必须同时活着」的前提。
- **可迁移**：双人产品演示——两个用户用同一产品协作完成任务。产品是「Drift 系统」——通过产品，两个陌生人变得同步。品牌叙事的「连接」主题——产品不是连接设备，而是「连接两个人」。
- **适用场景**：sci-fi, product-demo, studio-ad

### 怪兽分级制度（威胁升级引擎）

- **表现**：怪兽按体型/破坏力分为 1-5 级（Category I-V）。影片开始时，人类已经可以轻松击败 Cat I-II 级怪兽（用 3 代 Jaeger）。但很快 Cat III 出现（需要升级到 4 代 Jaeger）、然后是 Cat IV（4 代 Jaeger 也要苦战）、最后 Cat V（Slattern，需要核爆同归于尽）。每一级升级对应一次「人类以为够了但实际上不够」的打击。
- **手法**：威胁升级不是「打一个更强的」，而是「打一个让之前的方案全部失效的」。Cat I-II → 之前的 Jaeger 够用；Cat III → 之前的武器不够；Cat IV → 之前的策略不够（需要新战术）；Cat V → 之前的一切都不够（需要牺牲）。观众心理：「这次应该够了吧→还不够→还不够→天哪」。
- **可迁移**：产品功能演示的「功能阶梯」——功能 1 解决基础问题 → 功能 2 解决功能 1 暴露出的边缘问题 → 功能 3 解决功能 1+2 都无法处理的极端情况 → 核心功能 = 终极解决。每层不是「再加一个功能」，而是「让之前的功能方案失效了」。
- **适用场景**：product-demo, sci-fi

### 人类溃败 → 最后希望 → 胜利的三幕节奏

- **表现**：第一幕（溃败）：人类有 30+ 台 Jaeger，自信满满 → 被连续击败到只剩 4 台 → 政府决定放弃 Jaeger 计划，建「反怪兽墙」。第二幕（最后希望）：仅存的 4 台 Jaeger 被集中到香港基地 → 做最后的反击。第三幕（胜利）：用终极方案（核爆裂缝）堵住怪兽来源——但代价是所有 Jaeger 全毁，两个主角差点死掉。
- **手法**：不是「英雄从一开始就无敌」，而是「英雄被打到只剩最后一口气，最后用命拼赢」。第一幕的长度用于「让观众相信 Jaeger 很强」→ 然后在第一幕结尾「让观众看到最强的 Jaeger 被轻易撕碎」→ 绝望感建立。
- **可迁移**：品牌叙事中的「被质疑→证明→超越」结构——不是「产品一直很厉害」，而是「产品曾被质疑/被低估/被放弃→在绝境中证明了自己→最终不仅够用，而且超越了所有人的预期」。
- **适用场景**：sci-fi, studio-ad

### 同步叙事：驾驶员 → 机甲的三点跳转

- **表现**：驾驶舱内的两个驾驶员做同一个动作（同时挥拳/同时转身/同时踩下去）→ 机甲做这个动作 → 观众的视线在「人脸 A → 人脸 B → 机甲 → 回到人脸 A」之间循环跳跃。每次跳跃 1-2 秒，节奏极快。这个剪辑模式让观众同时看到了「人的努力」和「机器的威力」。
- **手法**：三点跳转剪辑：(1) 特写 Raleigh 的脸（发力表情）→ (2) 特写 Mako 的脸（同样发力）→ (3) 全景 Jaeger 做出动作 → (4) 中景两人同时喘息（动作完成后的释放）。一套跳转约 4-6 秒，每场战斗有 8-12 套跳转。
- **可迁移**：双人/团队产品的演示——两个用户分别在操作产品（分屏或快切）→ 产品界面/实物做出反应 → 两个用户的表情同步（一起笑/一起惊讶）。三点跳转让「产品效果」成为「人的反应」的证明。
- **适用场景**：product-demo, sci-fi

## 镜头语言（→ DP）

### 「重量感」的运动公式

- **表现**：Jaeger 的每次出拳不是标准动作片的 0.3-0.5s，而是 2-3s。慢加速（0.5-1s 达到峰值速度）+ 击中（冲击波 + 碎片）+ 长减速（1-2s 停下）+ 余震（机甲全身震荡 0.5s + 地面碎片持续掉落 1-2s）。每次动作后，画面中都有东西在「继续动」——碎片、雨水被震飞后下落、地面的余波。
- **技术参数**：动画缓动曲线用 ease-in-out-extreme：加速段 30%（慢 ramp up）→ 匀速段 20%（peak velocity）→ 减速段 30%（慢 ramp down）→ 余震 20%（overshoot + oscillation decay）。在 Blender/C4D 中：给大型物体的每个关键帧加「overshoot」——物体到达目标位置后会超过 5-10%，然后振荡 2-3 次回到目标位置。
- **可迁移**：产品 LOGO 演绎的「重量感」——LOGO 不是轻飘飘地飞入，而是像一块巨石一样「砸」到画面中：慢速进入 → 到达位置时画面震动 → LOGO 微微超过最终位置 5% → 回弹 → 停住 → 碎片/粒子继续飘落 2 秒。这比「快速飞入」传达的品牌分量高 10 倍。
- **适用场景**：logo-animation, sci-fi, product-demo

### 低角度仰拍（让 Jaeger 看起来巨大）

- **表现**：全片约 80% 的 Jaeger 镜头使用低角度（摄影机位置 ≈ 人的腰部高度或更低）。当 Jaeger 在画面中时，镜头从不高于其膝盖。观众永远在「仰视」这些巨型机甲——就像孩子仰视大人。
- **技术参数**：摄影机高度 = 1-3m（模拟一个站在机甲脚边的人仰望），广角（18-24mm）+ 向上倾斜 30-60°。关键：在画面底部保留人的参照物（一辆车/一个人/一个路灯）——只有和已知大小的物体对比，「巨大」才能被感知。
- **可迁移**：产品广告的低角度——手机放在地面上仰拍=手机看起来像建筑。LOGO 演绎：摄影机从 LOGO 底部仰角推上去 = LOGO 像摩天大楼。任何想让东西「看起来大」的场景=低角度+广角+前景参照物。
- **适用场景**：logo-animation, product-demo, sci-fi

### 驾驶舱内的人物调度（同步 = 情感同步）

- **表现**：驾驶舱的布局是两个人并排站立，相隔约 2m。当 Raleigh 和 Mako 的关系进展时，他们的「同步度」从视觉上表现出来：初期（陌生人）→ 两人动作有时差，一个先动另一个跟上；中期（拍档）→ 完全同步，但表情不同（Raleigh 专注、Mako 紧张）；后期（家人）→ 动作同步 + 表情同步 + 甚至呼吸节奏同步。
- **技术参数**：驾驶舱内用 2 机位（正面双人 + 侧面单人特写交替）。关键灯光：驾驶舱内只有头顶的全息投影做光源（冷暖交替），没有独立的面光——让角色的脸一直处在「半明半暗」中。
- **可迁移**：双人/团队产品的广告——两个人在不同空间使用同一产品，通过「同步动作」来暗示产品连接了他们。不需要他们对话——只靠动作同步就能传达「他们是一起的」。
- **适用场景**：product-demo, sci-fi, studio-ad

### 雨+水的体积感

- **表现**：全片所有关键战斗场景都在雨中/水中——不是因为导演喜欢雨，而是因为「水」是让巨型战斗看起来「有重量」的关键。(a) 雨滴打在 Jaeger 的金属表面=金属的质感被增强（湿金属比干金属有更多高光点）；(b) Jaeger 的脚踩进水里=水花溅起的体积和速度=力量的度量衡；(c) 雨水让光线散射=霓虹灯在雨中更美=画面更华丽。
- **技术参数**：每帧画面中约 1-3 万颗粒子（雨滴 + 水花 + 水雾）。雨滴有大（近景，可看到形状）中（中景，线条状）小（远景，雾状）三层。水下镜头：摄影机在水面以下，看到 Jaeger 和怪兽的脚从上方踩入水中——水面以上是战斗+雨水，水面以下是浑浊+碎片下沉。
- **可迁移**：产品广告的「下雨场景」——手表/手机/汽车上的雨滴 = 10 倍质感提升。每滴雨 = 一个微型透镜 = 折射产品 LOGO = 产品 LOGO 在每滴雨中都出现一次。产品 hero shot 的「水版本」：产品从水中升起来，表面挂着水珠——比干燥版本高级 3 倍。
- **适用场景**：sci-fi, product-demo, studio-ad

### 城市破坏的「地形变化」

- **表现**：Jaeger 和怪兽的战斗不是发生在「空地」上，而是发生在城市中——每次 Jaeger 被摔倒 = 压碎一条街的建筑，每次怪兽被甩出去 = 撞穿一座桥，每次双方角力 = 地面被踩出坑。城市不是「背景」，而是「战斗的第三个参与者」——城市的结构影响战斗（Jaeger 用集装箱当武器→用货船当棒子）。
- **技术参数**：破坏分三层：(1) 即时破坏——打击发生同时的碎片飞溅；(2) 延时破坏——被击中的建筑在 1-2s 后倒塌；(3) 环境后果——倒塌的建筑扬起烟尘/堵塞道路/改变地形（水从破裂的管道喷出、火从破裂的燃气管燃起）。不只是一次破坏就结束——每次打击后，环境「继续崩塌」。
- **可迁移**：产品演示的「环境互动」——不是产品在空白背景中，而是产品在使用时「改变」了周围的环境。手机放音乐→环境中的物体开始震动/跳舞。产品效果的可视化=不只看产品本身，看产品「改变了什么」。
- **适用场景**：sci-fi, product-demo

### 怪兽设计的「熟悉 + 陌生」原则

- **表现**：每只怪兽的身体部分来自真实动物，但组合方式不可能。Leatherback（棱背龟）= 大猩猩的上半身 + 龟壳 + 恐龙尾巴；Otachi（尾立鼠）= 蝙蝠翅膀 + 蛇的颈部 + 鳄鱼的嘴 + 喷酸液（地球上没有动物喷酸）；Slattern（毒妇）= 牛头 + 鞭尾 + 多根触手。观众潜意识「认识」每个部分（「这像猩猩」「这像蝙蝠」），但理性上知道这东西不存在。
- **设计流程**：(1) 选择 2-3 个真实动物作为「参考物种」；(2) 从每个参考物种取一个身体部分（头部/躯干/四肢/尾巴）；(3) 缩放各部分的比例（头部放大 3x/肢体缩短 50%）；(4) 加一个「不可能」的元素（喷酸/电磁脉冲/飞行）；(5) 统一材质（生物荧光皮肤 + 暗红色肌肉外露）。
- **可迁移**：产品设计的「熟悉+陌生」——概念产品的渲染：用熟悉的材质（金属/玻璃/布料）做陌生形态的产品。AI 生图 prompt：`giant creature design, kaiju, bioluminescent cyan blood, dark red organic tissue, combination of gorilla arms + turtle shell + dinosaur tail, Guillermo del Toro monster aesthetic`。
- **适用场景**：sci-fi, logo-animation

### Jaeger 的「国家文化」设计语言

- **表现**：每台 Jaeger 的设计融入其所属国家的文化元素。Gipsy Danger（美国）= 核反应堆在胸口（类似钢铁侠的方舟反应堆）+ 牛仔式站立姿态；Cherno Alpha（俄罗斯）= 粗壮的圆柱形头部（类似俄罗斯核电站冷却塔）+ 重型装甲（T-34 坦克基因）；Crimson Typhoon（中国）= 三条手臂（「三头六臂」的中国神话意象）+ 红色涂装 + 太极拳的动作风格；Striker Eureka（澳大利亚）= 流线型+速度型（袋鼠/猎犬的敏捷感）+ 胸口的导弹舱。
- **可迁移**：品牌的「本地化视觉」——为不同市场做产品视频时，产品的外形/涂装/动作风格融入当地文化元素（不是贴一个国旗就结束，而是产品的「动作方式」符合当地文化——中国 = 太极的圆转动作 / 美国 = 直来直去的拳击 / 日本 = 拔刀术的蓄力+爆发）。
- **适用场景**：product-demo, logo-animation

## 特效语言（→ VFX）

### 机械物理模拟（惯性 + 重量 + 地面冲击）

- **表现**：Jaeger 的动作不是「动画师手动调的」，而是用物理引擎模拟的——给定关节的目标角度 + 每个关节的质量/惯性/摩擦力/阻尼参数 → 物理引擎自动计算动作。(a) 出拳：肩膀关节给一个角速度 → 惯性让手臂继续摆 → 肘部关节在适当时机给反向力 → 拳头减速 → 击中目标 → 反弹力让整条手臂弹回来；(b) 走路：每步落地 → 脚底到膝盖到髋部的冲击波向上传递（用弹簧-阻尼模型模拟）→ 上半身微微后仰 → 0.5s 后回到平衡。
- **实现思路**：在 Houdini/Maya 中用 ragdoll physics + procedural animation。给 Jaeger 的每个关节设 mass/damping/spring stiffness → 用 IK（反向动力学）设目标位置 → 物理引擎 fill in 中间帧。关键参数：大型物体的 damping 要高（不容易停下来）、spring stiffness 要低（容易变形/晃动）。数字：Jaeger 的手臂质量约 200 吨，出拳的峰值速度约 150 km/h，动能 = ½ × 200,000kg × (41.7m/s)² ≈ 174 MJ（相当于 40kg TNT 的爆炸能量）。
- **可迁移**：Product demo 中的「物理真实感」——产品在广告中的运动不应是线性缓动，而应有物理模拟的惯性+弹跳+冲击。LOGO 演绎：LOGO 不是「飞入+停住」，而是「抛入+弹跳 2 次+停住」——因为有质量和惯性。
- **适用场景**：logo-animation, sci-fi, product-demo

### 雨水粒子系统（每帧数万粒子）

- **表现**：雨夜战斗场景中的雨水是太平洋的「视觉签名」。(a) 雨滴打在 Jaeger 表面→水花四溅；(b) Jaeger 快速挥拳→拳头前方的雨滴被气流推开→形成短暂的「无雨隧道」；(c) 怪兽被击中→蓝血喷出→蓝血与雨水混合→荧光青色的水花；(d) Jaeger 踩进水里→水面形成波纹扩散→波纹碰到建筑物反射回来→形成干涉图案。
- **实现思路**：多层级粒子系统（Houdini/Blender Mantaflow）。(1) 背景雨层：10000+粒子，均匀分布，下落速度 9m/s（终端速度），微风吹偏 5-10°；(2) 交互雨层：5000+粒子，与大型物体（Jaeger/怪兽）碰撞检测——碰到物体→变成水花粒子；(3) 水花粒子层：根据碰撞角度/速度生成——正面撞击=放射状水花，侧面擦过=切向水花；(4) 水雾层：体积粒子（不是点粒子），在物体快速移动后方的「尾流」中。
- **可迁移**：ComfyUI/Flux prompt：`heavy rain at night, neon-lit Hong Kong street, giant robot fighting in rain, rain particles colliding with metal surface, water splashes, cinematic volumetric lighting, blue-black atmosphere`。HyperFrames 中用 Canvas 粒子系统（约 2000 粒子即可）+ CSS 水花效果。
- **适用场景**：sci-fi, product-demo, studio-ad

### 驾驶舱全息 UI（HUD + 神经元同步）

- **表现**：驾驶舱内的全息投影 UI 有多个功能层：(a) 环境扫描——前方显示战场实时 3D 地图；(b) 机体状态——Jaeger 的每个部位颜色（绿=正常/黄=受损/红=严重/闪烁=即将失效）；(c) 神经元同步——显示两个驾驶员的脑波同步度（百分比），当同步度降到 70% 以下时 UI 变红闪烁；(d) 武器系统——显示可用武器和冷却时间。UI 是叙事的视觉化——观众通过 UI 就能知道「战况如何」。
- **实现思路**：在 AE/Blender 中做 holographic shader：(1) 基础几何体（线框模式）；(2) 叠加 scanline（横向条纹，模拟全息扫描线，opacity 30%）；(3) 边缘发光（fresnel glow, 蓝青色）；(4) 微闪烁（opacity oscillation, 2-3Hz, 幅度 5%）；(5) 色彩偏移（chromatic aberration, RGB 通道 offset 1-2px）。
- **可迁移**：产品广告的 UI 设计——产品配套 APP 的界面不应该「完美」，应该有轻微的 scanline/flicker——让科技感「真实」。HUD 动画模板：线框旋转 + scanline + fresnel glow + micro flicker = 任何数据都看起来像「科幻界面」。
- **适用场景**：sci-fi, product-demo, logo-animation

## 声音设计（→ Sound Designer）

### 关节专属机械音（每个关节有独立音色）

- **表现**：每台 Jaeger 的每个关节有专属声音——肩膀旋转 = 重型液压声（低频 60-80Hz），肘部弯曲 = 齿轮咬合声（中频 200-500Hz），拳头握紧 = 金属应力声（高频 2-4kHz），膝关节落地 = 冲击吸收声（低频+中频混合）。不同 Jaeger 的关节声音也不同——Gipsy Danger 的更「流畅」（液压为主），Cherno Alpha 的更「粗暴」（齿轮撞击为主）。
- **声音层次**：机械音 (40%) > 雨声 (25%) > 怪兽音 (20%) > 人声 (10%) > 音乐 (5%)。这是一个「反常规」的混音——大多数动作片音乐占 60-70%，但 Pacific Rim 把音乐压到最低，用机械音+雨声做「音乐」。
- **可迁移**：产品广告的「机械质感音」——相机广告中快门/镜头对焦的声音、汽车广告中引擎启动和换挡的声音、手表广告中机械机芯的滴答声。每种声音的 pitch/timbre/rhythm 都要独一无二——不是「一个音效」，而是「一套声音系统」。
- **适用场景**：sci-fi, product-demo, studio-ad

### 怪兽吼叫的「多层叠加」

- **表现**：每只怪兽的吼叫不是单一动物的录音，而是 3-5 种动物的声音叠加。Leatherback 的吼叫 = 大象的低频吼（30-50Hz 次声波感）+ 鳄鱼的喉音（中低频 100-200Hz）+ 猪的尖叫声（高频 2-3kHz 刺耳）+ 金属扭曲声（4-8kHz 不和谐的泛音）。多层叠加让怪兽声音既「熟悉」（能听出是动物）又「不可能」（现实中不存在这种混合）。
- **技术参数**：(1) 选择 3-5 个动物声音源；(2) 将每个声音的 pitch 调整到不同频段（大象 0.5x speed=低至 15Hz, 猪 2x speed=高至 5kHz）；(3) 在 DAW 中分层，每层的 attack/release 不同（低频层 slow attack + long release, 高频层 fast attack + short release）；(4) 加 distortion/saturation 让声音「脏」；(5) 加 subharmonic synthesizer 在 20-40Hz 产生「感觉不到但听得到」的次声波。
- **可迁移**：产品音效设计——手机振动声、电脑启动声、汽车引擎声——每个声音也应该是「多层叠加」的而非单一频率。品牌声音 LOGO（如 Netflix 的「tudum」）也可以用多层叠加设计——不是你听到的那么简单。
- **适用场景**：sci-fi, logo-animation

### 雨中战斗的三层声音深度

- **表现**：雨夜战斗的声音有三层深度。(1) 近景层（0-50m）：Jaeger 的关节机械音+怪兽的呼吸声+驾驶员的对话——清晰、响亮、无混响；(2) 中景层（50-200m）：雨声+水花声+建筑碎片掉落声——中等音量+轻微混响（模拟雨水和建筑物的声音扩散）；(3) 远景层（200m+）：远处其他 Jaeger 的战斗声+城市背景声（警报/爆炸）+雷声——低音量+长混响+低通滤波（高频被雨水吸收）。这三层声音让观众能「听到空间」——不只是「看到」Jaeger 在雨中战斗，而是「听到」战斗的空间尺度。
- **可迁移**：任何「户外场景」的广告声音都应该有三层深度——近（产品的声音/人的对话）、中（环境音/周围人的活动声）、远（城市背景/自然声）。三层混音让视频从「2D 声音」变成「3D 声音」。
- **适用场景**：sci-fi, studio-ad, product-demo

### 电吉他 + 交响乐的主题音乐

- **表现**：Ramin Djawadi 的主题音乐（就是《权力的游戏》的作曲家）用电吉他的 heavy riff（低音弦的强力下拨）代表 Jaeger 的力量和攻击性，用交响乐团（铜管+弦乐）代表战斗的史诗感。关键设计：电吉他不是背景——电吉他是「领奏乐器」，交响乐是「伴奏」。这在传统好莱坞配乐中是罕见的（传统是交响乐为主、摇滚为辅）。
- **声音层次**：电吉他 heavy riff（40-60% 段落，主导） > 铜管/弦乐（全程，衬底） > 打击乐/鼓（战斗段落，推进节奏） > 合成器 pad（过渡段落，氛围）。
- **可迁移**：品牌广告的配乐策略——找一个「不该出现在这个类型」的乐器做领奏（古筝+电子乐 / 二胡+摇滚 / 笛子+trap beat）。反差=记忆点。AI 视频的 BGM：用 Suno/Udio 生成时指定「electric guitar riff + epic orchestra + industrial percussion」。
- **适用场景**：sci-fi, studio-ad, product-demo

## 适用提示

- **最佳匹配场景**：需要「巨型机械/重量感/物理真实」的科幻或产品项目。需要「雨夜/霓虹/战斗」的视觉华丽场景。需要「同步/协作/连接」主题的双人/团队品牌叙事。
- **不适用场景**：温馨家庭广告、极简设计美学（Pacific Rim 的核心是「满」不是「简」）、不需要物理重量感的轻量产品。
- **引用优先级**：核心引用（做巨型机械/机甲项目时——重量感+雨水+机械音三者同时需要）；特定技法借用（仅需雨水粒子时，只看 VFX 段；仅需低角度仰拍时，只看镜头语言段）。
